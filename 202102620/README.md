## 퀴즈 정답 (Git-Flow 기준, 강의 자료)

1. **새로운 작업이 주어졌을 때 사용하는 브랜치**  
   **feature 브랜치**입니다. `develop`에서 분기해 다음 릴리스를 위한 기능을 개발합니다.

2. **작업 완료 후 PR을 어디로 머지 요청할까**  
   일반적인 기능 개발이라면 **develop**으로 Pull Request를 보냅니다. (릴리스 준비·배포 단계에서는 `release` → `master` 등 단계가 달라질 수 있습니다.)

3. **이미 master에 머지된 작업에 긴급 수정이 필요할 때**  
   **hotfix 브랜치**를 씁니다. 운영(`master`)에서 바로 문제를 고치기 위해 `master`에서 분기하고, 수정 후 `master`와 `develop`(또는 진행 중인 `release`)에 반영합니다.
